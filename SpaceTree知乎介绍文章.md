# 空间树 SpaceTree：一款被名字耽误的磁盘文件管理工具

> 2026-07-13 · 基于 v3.50.0 实测体验 + 源码审读

我磁盘里有多少重复文件？这个问题大多数人都答不上来。

摄影师攒了几年外拍，同一个 RAW 原片可能存了三四份在不同目录里；程序员的 `node_modules` 和编译产物在不同项目间大量重复；下载党硬盘里塞满了同名不同次的视频文件。空间就这么一点点被吃掉的。

最近在 GitHub 上发现一个叫"空间树 SpaceTree"的开源工具，名字听起来像磁盘空间分析软件，但实际用下来发现它的功能比名字暗示的多得多。我读了它的完整源码，确认了功能不是吹的，写这篇文章记录一下。

---

## 它能做什么

一句话概括：**扫描你的磁盘，把所有文件建库索引，然后在这之上做重复检测、去重清理、目录浏览、相似图片查找等一整套文件管理操作。**

重点不是"扫描重复文件"——这只是它众多功能之一。它的核心是"建库"，即把磁盘上所有文件的元数据（路径、大小、哈希、时间戳等）存进本地 SQLite 数据库，后续所有操作都在这个数据库上进行，不用每次重新扫磁盘。

安装包只有 5.5MB，开源免费，MIT 协议。

---

## 值得说的几个功能

### 目录树与文件建库：被严重低估的功能

这是我用下来觉得最有价值，但 README 里几乎没怎么提的功能。

扫描完成后，所有文件的元数据都存在本地数据库里。你可以在工具内以目录树的形式浏览整个磁盘结构，左边点目录，右边出文件列表，跟资源管理器类似但有一个关键区别：**它支持跨目录搜索**。在搜索框输入关键词，瞬间在所有已扫描的文件里查找，不用像资源管理器那样一个目录一个目录地翻。

更实用的是，这个数据库是持久化的。你上周扫描过一次，这周想查"上周 D 盘照片目录里有哪些文件"，直接查库就行，不用重新扫描。配合增量扫描功能（后面会讲），二次扫描只处理新增和修改的文件，几十万文件的盘几分钟就能更新完。

说白了，它把"扫一次用一次"的清理工具，变成了"扫一次反复查"的文件资产管理工具。

### 增量扫描：第二次扫描快很多

第一次全量扫描会计算每个文件的哈希，确实需要一些时间。但第二次扫描时，增量扫描功能会拿数据库里的已有记录跟当前文件系统做对比：文件大小没变就跳过，大小变了才重新算哈希，数据库里有但磁盘上没了就标记删除。

实际效果是：二次扫描的时间远小于首次全量扫描，因为大部分文件没动过，直接跳过了。对于几十万文件的磁盘，这个功能的价值很明显。

### 硬链接去重：释放空间但不破坏目录结构

重复文件清理的常见痛点是：删了重复文件，但你的目录结构也被破坏了。比如照片分类夹里存了一份副本，项目目录里又存了一份，删掉任何一个都会影响对应目录的完整性。

硬链接去重解决的就是这个问题。它把重复文件替换成指向同一个物理数据的硬链接——目录结构不变，文件还在原来的位置，但磁盘上只存一份数据，多出来的空间被释放了。

但这里有个关键问题：**硬链接替换如果中途失败，文件会不会丢？**

我特意去读了源码确认这一点。它的实现是事务型的——先把待替换文件重命名为 `.dfh_temp` 备份，再创建硬链接，链接成功就删掉备份，链接失败就把备份重命名回原路径。最坏情况是你磁盘上多一个 `.dfh_temp` 文件，原始数据不会丢。

### 删除前的安全审计：怕误删可以看看这里

很多人对"自动清理重复文件"有顾虑：万一判断错了，把不重复的文件删了怎么办？

这个工具在删除前做了一道安全审计。扫描阶段用的是文件头尾采样哈希（速度快但有理论碰撞可能），删除前会对保留文件和待删除文件做一次**全文件完整哈希比对**，不一致就弹窗拦截，不会执行删除。

也就是说，即使扫描阶段把两个内容不同的文件误判为重复，删除前的全文件校验也会拦住。这是一个"快速初筛 + 严格终审"的双层设计。

### 相似图片查找：不是找完全相同，是找"看起来像"

这个功能跟重复文件检测不同。重复检测是哈希完全一致，相似图片查找是感知哈希（pHash）的汉明距离在阈值以内——即图片不是完全一样，但视觉上看起来很像。

典型场景：同一张照片做了不同参数的后期处理，存了几个版本；或者截图截了两次，内容几乎一样但尺寸略有不同。这些用文件哈希找不到，但用感知哈希能匹配到。

实现上用了 BK-Tree 数据结构做近邻搜索，比暴力两两比对快很多。源码里 BK-Tree 的插入和搜索都是手写实现的，包括汉明距离剪枝逻辑。

### 矩形树图：一眼看清空间都被谁占了

分析中心有一个可视化的矩形树图，每个矩形代表一个重复文件组，面积越大表示浪费的空间越多。支持滚轮缩放和拖拽平移，双击可以下钻到文件所在目录。

技术上用的是 Bruls 等人的 Squarified Treemap 算法（手写实现，不是调库），确保矩形宽高比尽量接近正方形。在 4K 屏上也能清晰显示，做了 DPI 自适应。

## 其他功能一览

| 功能 | 说明 |
|------|------|
| 文件搜索 | 按扩展名、大小、日期范围筛选 |

## 技术栈

Tauri v2（Rust 后端 + WebView 前端），Vue 3 + TypeScript + Element Plus，SQLite 本地存储。安装包 5.5MB，比 Electron 方案小一个数量级。

性能相关的技术选型：Rayon 多线程并行扫描、Crossbeam 无锁通道做线程间数据传递、XXH3 作为默认哈希算法（速度优先），同时支持 BLAKE3/SHA-256 等加密哈希。

---

## 目前的局限

说几个需要注意的点：

1. **只有 Windows 安装包**。CI 只构建 Windows 平台，macOS 和 Linux 用户需要自己从源码编译。
2. **项目比较新**。2026 年 4 月才发布 v1.0，到现在三个多月。功能迭代很快，但稳定性还需要更多用户反馈来验证。
3. **硬链接不能跨分区**。这是 NTFS 的限制不是软件的问题，但使用时要注意保留文件和待替换文件必须在同一个盘。
4. **暂无自动化测试**。源码里没有单元测试和集成测试，可靠性目前靠手动验证。希望后续能补上。
